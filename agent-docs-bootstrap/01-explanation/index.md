# Explanation

Understanding-oriented documentation helps readers comprehend the system — its purpose, how it works, and why it is designed that way.

## Contents

- [Architecture](architecture.md) — System structure and components
- [Design Decisions](decisions.md) — Rationale behind key choices

## When to Write Explanation Docs

- Introduce a concept or feature
- Explain *why* something works the way it does
- Provide background knowledge needed to understand other docs
- Discuss trade-offs and alternatives considered

## Style Guidelines

- Use narrative prose, not bullet steps
- Address the reader directly ("you")
- Avoid instructing ("do this") in favor of describing ("this works")
- Provide context without assuming expertise
