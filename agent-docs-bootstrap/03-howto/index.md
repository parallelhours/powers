# How-To Guides

Task-oriented documentation that helps readers achieve specific goals.

## Available Guides

- [Installation](installation.md) — Install and configure the project
- [Configuration](configuration.md) — Configure settings and options
- [Set Up Issue Labels](setup-issue-labels.md) — Configure agile labels in GitHub, Jira, or parallelhours.io

## When to Write How-To Docs

- Guide readers through a specific task
- Solve a particular problem or use case
- Cover common workflows

## Style Guidelines

- Use numbered steps for sequences
- Include code examples with expected output
- Provide troubleshooting within each guide if common issues exist
- Be goal-oriented, not conceptual
