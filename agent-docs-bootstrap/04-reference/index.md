## When to Write Reference Docs

- Document APIs, CLIs, configurations, or schemas
- Provide exhaustive detail
- Enable lookup rather than learning

## Style Guidelines

- Be complete and accurate
- Use clear, consistent structure
- Avoid explanation or context — just facts
- Use tables, code blocks, and lists
