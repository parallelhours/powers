# Glossary

Terms and definitions used in this project.

## A

**API**
: Application Programming Interface. A set of protocols for building software applications.

## C

**CLI**
: Command-Line Interface. Text-based interface for interacting with software.

## D

**Diátaxis**
: Documentation framework that defines four documentation types: tutorials, how-to guides, explanation, and reference. Pronounced "dee-ah-TAX-iss."

## G

**Gantt Chart**
: A bar chart that illustrates a project schedule, showing tasks against time. Also called a timeline view or schedule bar chart.

## I

**Iteration**
: Another term for a sprint or time-boxed development cycle. See Sprint.

## E

**Environment Variable**
: Variable that affects how processes run on a system.

## M

**Middleware**
: Software that acts as a bridge between an operating system or database and applications.

## S

**Schema**
: Structure that defines the organization of data.

**Sprint**
: A time-boxed development cycle, typically 1-2 weeks, following Scrum/agile methodology. Also called an iteration or cycle.
