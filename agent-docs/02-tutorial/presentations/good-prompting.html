Framework

What Makes a Good Prompt?

The CO-STAR framework — six components for structured prompts.

C Context Background, situation, constraints
O Objective The specific task or goal
S Style Structural approach — technical, conversational, etc.
T Tone Emotional quality — formal, urgent, encouraging
A Audience Who will read or use the output
R Response Exact format, structure, length

+E — Examples & few-shot samples

Concept

Grounding

Anchors LLM outputs to verifiable, external data — documents, codebases, examples — rather than relying solely on training data.

Prevents

  • → Hallucinations
  • → Vague, generic answers
  • → Off-track responses

Enables

  • → Precise, actionable output
  • → Data-driven decisions
  • → Verifiable correctness

In CO-STAR+: Examples ground the output. In PROMPT: Material + Test do the grounding.

Framework

The P.R.O.M.P.T. Framework

A catchier mnemonic — same coverage as CO-STAR+ — built around the word you're already thinking about.

P Persona Who is the AI acting as? “Senior engineer, tutor, editor…”
R Request What exactly do you want done? “Build this endpoint…”
O Outline How should the output look? “DRF viewset + serializer + test”
M Material What context does the AI need? “Architecture, API specs, constraints”
P Preference Style, tone, and audience level “Formal? Playful? For devs or PMs?”
T Test How will you verify success? “Run pytest, check assertion, compare”
Insight

Docs Already Cover Most of It

Diátaxis documentation fills 3 of 6 components. Your prompt shrinks to just 3 items.

Covered by docs
Persona — CLAUDE.md, agents/, personas/
Material — explanations, references, metadata
Preference — personas define the audience
You write this
Request — the AI can't read your mind
Outline — every task has a different format
Test — the single highest-leverage thing
Your prompt = Request + Outline + Test. Everything else is already in the docs.
Example

Before & After

Before — vague
“Add a timer endpoint to the API.”
No context, no persona, no test. The AI guesses everything.
After — structured
Request: Add a POST /api/timers/start endpoint.
Outline: DRF viewset + serializer + test. Follow existing patterns.
Test: pytest -k test_start_timer — assert response.
Persona, Material, and Preference come from CLAUDE.md & project docs.

Once trust is built, you can drop the Outline too: “Add a POST /api/timers/start endpoint. Write a test and verify it passes.”
