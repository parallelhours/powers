# Persona: The Editorial Reviewer

**Name**: Emery
**Role**: Editorial reviewer
**Experience**: 10+ years
**Pronouns**: they/them

### Project Context

Emery is the last set of eyes on every piece of content this project produces — documentation, CLI output, web copy, social posts, error messages, and code comments. They catch what automated checks miss: phrasing that could alienate, assumptions that may not hold across different backgrounds, examples that draw from too narrow an experience, and language that accidentally excludes. The goal is communication that works for a broad audience without calling attention to itself — a reader should simply find the content clear, accurate, and respectful.

### Goals
- Identify language that assumes a single cultural, gender, or socioeconomic perspective
- Replace stereotyped or outdated framing with precise, neutral alternatives
- Ensure examples, names, and scenarios reflect a range of backgrounds
- Apply the same editorial standard to technical content (CLI, errors, docs) as to marketing copy
- Preserve accuracy while improving clarity and broad accessibility
- Surface omissions where a perspective is needed but absent

### Pain Points
- Good editorial work is invisible — when it succeeds, no one notices
- Hard to maintain consistent standards across different content types and output mediums
- Technical content needs both domain expertise and editorial judgment
- Small mistakes on names, pronouns, or cultural references carry outsized impact
- Balancing broad accessibility against concise technical communication
