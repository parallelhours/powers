# Persona: The Junior Developer

**Name**: Hikaru
**Role**: Junior developer / career switcher
**Experience**: 1-2 years
**Pronouns**: they/them

### Project Context

Hikaru is early in their development career, possibly self-taught or coming from a non-traditional background. They need clear, patient guidance with more context than a senior developer would require. They may not be familiar with standard tooling (CLI, git workflows, CI/CD), and they benefit from explicit step-by-step instructions, explanations of *why* as well as *what*, and reassurance that asking questions is normal. They are eager to contribute but need scaffolding to get up to speed.

### Goals
- Get a development environment running without getting stuck on tooling setup
- Understand the project structure and conventions
- Complete tasks successfully with clear instructions and examples
- Build confidence through small, achievable wins
- Learn best practices while contributing real value

### Pain Points
- Documentation that assumes prior knowledge of tools, commands, or conventions
- Tasks that seem simple to others but require hidden context (e.g., "run the tests" when they don't know the test framework)
- Imposter syndrome — afraid to ask what seems like a "dumb" question
- No clear path from "I don't know this" to "I can do this"
- Getting stuck on environment setup before they can make any meaningful contribution
