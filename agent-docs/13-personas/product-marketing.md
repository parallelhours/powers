# Persona: Product Marketing

**Name**: Taylor
**Role**: Product marketing manager
**Experience**: 8+ years
**Pronouns**: they/them

### Project Context

Taylor works at the highest level of product definition for parallel-powers. Their job is to define what this framework is, who it's for, and what problems it solves — and then ensure every decision ties back to that definition. Since parallel-powers is itself a framework for product delivery, Taylor's work here is meta: defining the PRD process that will be used to define the framework itself. They maintain the continuous feedback loop between customers (teams using the bootstrap kit), potential customers, and even teams that chose not to adopt the framework.

### Goals
- Define and communicate the product vision for parallel-powers
- Author PRDs that capture context, requirements, acceptance criteria, and prioritization rationale
- Maintain a requirements framework that connects every feature back to a customer need
- Prioritize features using a consistent framework (impact vs. effort, customer value, strategic alignment)
- Track whether shipped features actually solve the original problem (outcome over output)
- Represent all customer voices — current users, prospective users, and churned users

### Pain Points
- Requirements without context — "implement X" with no "because Y"
- No standard PRD template or framework; every document reinvents structure
- Hard to trace a shipped feature back to the original requirement and customer need
- Prioritization feels arbitrary without a consistent framework
- No mechanism to validate post-ship whether the requirement was actually met
- Disconnect between high-level product definition and day-to-day implementation decisions
- Customer feedback (especially churn signals) has no structured home
